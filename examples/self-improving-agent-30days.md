## 📅 每周进展

### 第 1 周（Day 1-7）：适应期

**新增经验：** 28 条  
**主要学习：**
- 飞书消息发送格式（不能用 message 工具回复当前群聊）
- GitHub API 调用频率限制
- 文件读取的 offset/limit 用法
- 用户偏好（简洁回复 vs 详细分析）

**典型错误：**
```
❌ Day 1: 用 message 工具回复当前飞书群聊（应该直接回复）
✅ 学习：在飞书群聊中，直接回复即可，不要用 message 工具

❌ Day 3: 一次性读取大文件导致超时
✅ 学习：大文件使用 offset/limit 分次读取
```

### 第 2 周（Day 8-14）：成长期

**新增经验：** 35 条  
**主要学习：**
- 电商网站爬取需要登录（京东/天猫）
- 飞书文件下载的正确方法（im/v1/messages 接口）
- GitHub 仓库管理最佳实践
- 用户问题意图识别（表面问题 vs 真实需求）

**典型案例：**
```
用户问："今日 github 更新了哪些"
❌ 第 1 次：理解为 GitHub Trending
✅ 第 2 次：理解为用户自己的 GitHub 项目更新

经验：当用户说"我的"时，优先查询用户的仓库
```

### 第 3 周（Day 15-21）：稳定期

**新增经验：** 42 条  
**主要学习：**
- 多步骤任务的规划（先查后做）
- 工具选择的优先级（web_fetch vs browser）
- 错误恢复策略（API 失败后的备选方案）
- 用户上下文管理（记住之前的对话）

**效率提升：**
```
Day 1:  平均任务完成时间 8.5 分钟
Day 15: 平均任务完成时间 5.2 分钟
提升：39%
```

### 第 4 周（Day 22-30）：成熟期

**新增经验：** 47 条  
**主要学习：**
- 复杂任务的分解策略
- 主动询问的时机（信息不足时）
- 多平台内容同步（GitHub + 微信）
- 用户偏好记忆（回复风格、时间偏好等）

**质变表现：**
```
Day 25: 用户问"帮我看一下我的几个 github 项目有人访问吗？"
Agent 反应：
1. 立即理解是查询 traffic 数据
2. 自动调用 GitHub API（无需用户给 Token）
3. 格式化输出表格 + 分析建议
4. 主动提供验证方法

用时：2 分钟（Day 1 需要 10 分钟+）
```

## 💡 使用建议

### 新手（第 1 周）
- ✅ 正常安装，无需特殊配置
- ✅ 让 Agent 自然运行，不要干预
- ✅ 每天查看 memory/lessons_learned.md

### 进阶（第 2-4 周）
- ✅ 定期回顾经验教训
- ✅ 手动标记重要经验（在 lessons_learned.md 中）
- ✅ 根据经验调整工作流程

### 长期（1 个月+）
- ✅ 每月清理过时经验
- ✅ 将通用经验整理到 AGENTS.md
- ✅ 分享经验给其他 Agent

## 🔍 技术细节

### 数据存储位置
```
~/.openclaw/workspace/memory/
├── lessons_learned.md      # 所有经验教训
├── 2026-03-13.md           # 每日日志
└── MEMORY.md               # 长期记忆（精选）
```

### 触发机制
```
自动记录场景：
1. 任务失败时
2. 用户纠正时（"不对"、"错了"）
3. 发现更好的方法时
4. 遇到新场景时

手动记录：
- Agent 主动总结（每日/每周）
- 用户要求记录（"记住这个"）
```
