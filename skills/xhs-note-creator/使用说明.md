# 📤 小红书发布技能 - 使用说明

## ✅ 已优化功能

### 渲染脚本 (`render_xhs_v2.js`)

1. **封面优化**
   - 标题支持换行（在 Markdown YAML 中使用 `\n` 分隔）
   - 标题居中显示，更美观
   - 副标题可选，自动适配

2. **正文卡片**
   - 智能分页，自动计算内容高度
   - 支持多种样式主题
   - 页码显示（多页时）

3. **文案生成**
   - 自动生成 `caption.txt`
   - 包含标题、副标题、正文摘要、话题标签
   - 自动去重标签

### 发布脚本 (`publish_xhs.py`)

1. **智能文案读取**
   - 自动查找并读取 `caption.txt`
   - 支持手动指定 caption 文件路径
   - 支持命令行直接指定描述

2. **错误处理**
   - 详细的错误提示和排查建议
   - Cookie 验证
   - 图片文件验证

3. **Dry-run 模式**
   - 发布前验证所有参数
   - 不实际发布，仅检查

## 📖 使用流程

### 1️⃣ 准备 Markdown 文件

将草稿放在 `content/` 或 `drafts/` 目录：

```bash
# 新建草稿
vim drafts/draft_new_topic.md

# 完成后移动到 content/
mv drafts/draft_new_topic.md content/content_new_topic.md
```

**Markdown 格式：**
```markdown
---
emoji: "🤖"
title: "标题第一行\n标题第二行"
subtitle: "副标题文案"
---

# 正文内容

正文段落 1
---
正文段落 2

# 话题标签
#OpenClaw #AI #自动化
```

### 2️⃣ 渲染图片

```bash
cd /home/admin/.openclaw/workspace/skills/xhs-note-creator

# 方式 1：自动创建时间戳目录（推荐）
node scripts/render_xhs_v2.js content/content_new_topic.md

# 方式 2：指定输出目录
node scripts/render_xhs_v2.js content/content_new_topic.md -o ./output/2026-03-05-topic

# 方式 3：指定样式
node scripts/render_xhs_v2.js content/content_new_topic.md -s xiaohongshu

# 查看可用样式
node scripts/render_xhs_v2.js --list-styles
```

**输出：**
```
  📁 从 content 目录找到：.../content/content_new_topic.md
  📂 输出目录：./output/2026-03-05_12-38-45
✨ 渲染完成！共生成 4 张图片（1 封面 + 3 正文）
  📝 文案已保存：./output/2026-03-05_12-38-45/caption.txt
```

### 3️⃣ 发布笔记

```bash
# 进入输出目录
cd output/2026-03-05_12-38-45/

# 发布（自动读取 caption.txt）
source ../../.venv/bin/activate
python ../../scripts/publish_xhs.py -t "笔记标题" -i cover.png card_*.png

# 或者使用绝对路径
python scripts/publish_xhs.py -t "标题" -i /path/to/output/cover.png /path/to/card_*.png
```

## 🔧 环境配置

### 依赖安装

```bash
cd /home/admin/.openclaw/workspace/skills/xhs-note-creator

# Node.js 依赖（渲染图片）
pnpm install

# Python 依赖（发布笔记）
uv venv --python 3.14
source .venv/bin/activate
uv pip install xhs python-dotenv requests
```

### Cookie 配置

创建 `.env` 文件：

```bash
# .env
XHS_COOKIE=acw_tc=...; a1=...; web_session=...; ...
```

**获取 Cookie：**
1. 浏览器登录小红书（https://www.xiaohongshu.com）
2. F12 打开开发者工具
3. Network 标签查看任意请求的 Cookie
4. 复制完整字符串

## 💡 最佳实践

### 1. 封面标题格式

```yaml
# 推荐：使用 \n 换行
title: "主标题第一行\n主标题第二行"

# 避免：过长的单行标题
title: "这是一个非常非常非常长的标题超过了 20 字"
```

### 2. 正文卡片数量（重要！）

**自动限制为 3 张**，更多内容会自动放入文案：

- ✅ 封面 1 张 + 正文 2-3 张
- ✅ 文案包含完整内容（5-8 段）
- ✅ 用户滑动图片压力小
- ✅ 文案信息量大，利于 SEO

**内容组织技巧：**
- 前 3 个内容块放核心要点（会渲染成图片）
- 后续内容块放详细说明（会放入文案）
- 使用 `---` 合理分隔不同主题

### 3. 文案优化

- 标题：不超过 20 字（支持换行）
- 副标题：补充说明（可选）
- 描述：前 5-8 段自动提取
- 标签：5-15 个相关话题（自动去重）

### 4. 发布时机

- 工作日：早 8-9 点、午 12-13 点、晚 20-22 点
- 周末：上午 10-12 点、下午 15-17 点

## ❓ 常见问题

### Q: 渲染失败 "Executable doesn't exist"
**A:** 确保已安装 Playwright 浏览器：
```bash
pnpm exec playwright install chromium
```

### Q: 发布失败 "签名错误"
**A:** Cookie 可能过期，重新获取 Cookie

### Q: 文案为空
**A:** 确保 `caption.txt` 在输出目录，或手动指定 `-c` 参数

### Q: 图片上传失败
**A:** 检查图片格式（PNG/JPG）和大小（<10MB）
