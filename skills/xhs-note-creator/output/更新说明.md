# 技能优化更新说明

## 🐛 修复的问题

### 1. Tags 标签格式错误
**问题**：之前 tags 带有 `#` 前缀（如 `#OpenClaw #AI 助理`），不符合小红书格式要求

**修复**：
- SKILL.md 中明确要求 tags 使用**空格分隔的纯文本**
- 发布脚本新增自动提取 tags 功能，会自动移除 `#` 前缀
- 示例：`OpenClaw AI 助理 自动化 效率工具`

### 2. 文案换行丢失
**问题**：发布的文案换行符没有正确保留，导致格式混乱

**修复**：
- 发布脚本新增 `normalize_desc()` 函数，确保换行符正确处理
- 支持 `\\n` 转义序列自动转换为实际换行
- 从 Markdown 文件发布时，自动保留原始排版

## ✨ 新增功能

### 从 Markdown 文件发布（推荐）

现在可以直接从 Markdown 文件发布，脚本会自动：
1. 提取 YAML frontmatter 之后的正文
2. 提取 `# 🏷️` 部分的 tags（自动移除 `#` 前缀）
3. 移除 `# 🏷️` 部分，避免出现在最终文案中
4. 自动追加 tags 到文案末尾

**用法**：
```bash
python scripts/publish_xhs.py --title "笔记标题" --md content.md --images cover.png card_1.png
```

## 📝 正确的 Markdown 格式示例

```markdown
---
emoji: "🚀"
title: "大标题"
subtitle: "副标题"
---

# 正文内容

这里是正文段落，使用空行分隔。

---

# 🏷️

标签 1 标签 2 标签 3 标签 4
```

**注意**：
- Tags 用**空格分隔**，不要加 `#` 前缀
- 每段之间用**空行**分隔
- 使用 `---` 分割不同的卡片内容

## 🔄 如何更新

技能已自动更新，下次发布时使用新参数即可：

```bash
# 推荐：从 Markdown 发布
python scripts/publish_xhs.py -t "标题" --md content.md -i cover.png card_1.png

# 或者继续手动指定描述
python scripts/publish_xhs.py -t "标题" -d "描述内容" -i cover.png card_1.png
```
