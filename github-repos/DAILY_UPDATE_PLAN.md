# 📅 OpenClaw 项目矩阵每日更新计划

> 三个核心项目协同运营，最大化影响力

---

## 🎯 项目定位

| 项目 | 定位 | 目标用户 | 更新频率 |
|------|------|---------|---------|
| **openclaw-recommended-skills** | Skill 推荐清单 | 所有用户 | **每日** |
| **openclaw-starter** | 场景化配置模板 | 新手/进阶 | 每周 3 次 |
| **openclaw-workflows** | 自动化工作流 | 进阶用户 | 每周 2 次 |

---

## 📆 每周更新计划

### 周一：新 Skill 推荐日

**openclaw-recommended-skills:**
- ✅ 检查 ClawHub 新上架 Skill
- ✅ 测试并评估
- ✅ 更新推荐列表
- ✅ 发布更新日志

**协同更新:**
- openclaw-starter: 如适用，添加到场景配置
- openclaw-workflows: 如适用，集成到工作流

---

### 周二：技巧分享日

**openclaw-recommended-skills:**
- ✅ 发布 Day XXX 使用技巧
- ✅ 代码示例
- ✅ 最佳实践

**示例内容:**
```markdown
## Day 001 - 使用 searxng 隐私搜索

```bash
openclaw agent -m "搜索 Python 最新教程"
```

💡 技巧：使用 `--session-id` 避免上下文干扰
```

---

### 周三：场景配置日

**openclaw-starter:**
- ✅ 新增/优化场景配置
- ✅ 更新使用说明
- ✅ 提供配置对比

**协同更新:**
- openclaw-recommended-skills: 推荐相关 Skill
- openclaw-workflows: 提供配套工作流

---

### 周四：工作流分享日

**openclaw-workflows:**
- ✅ 发布新工作流
- ✅ 详细配置步骤
- ✅ 使用示例

**示例内容:**
```markdown
## 每日简报工作流

功能：每天早上 8 点自动汇总新闻、天气、日程

所需 Skill: searxng, tavily-search, summarize

配置步骤：
1. 创建 cron 任务
2. 配置通知渠道
3. 测试运行
```

---

### 周五：用户问答日

**三个项目协同:**
- ✅ 回答本周用户问题
- ✅ 整理常见问题
- ✅ 更新 FAQ

**发布平台:**
- GitHub Issues
- README FAQ 章节
- 社交媒体

---

### 周六：案例分享日

**轮流更新:**
- Week 1: openclaw-recommended-skills - Skill 使用案例
- Week 2: openclaw-starter - 场景配置案例
- Week 3: openclaw-workflows - 工作流应用案例

---

### 周日：周总结日

**三个项目协同:**
- ✅ 本周更新总结
- ✅ 数据统计（Stars/Forks/Issues）
- ✅ 下周计划预告

---

## 📊 每日工作流程

### 上午（9:00-10:00）

1. **检查数据** (5 分钟)
   - 查看昨日 Stars 增长
   - 检查新 Issues/PRs
   - 回复紧急问题

2. **内容准备** (30 分钟)
   - 根据今日主题准备内容
   - 编写 README 更新
   - 准备代码示例

3. **发布更新** (15 分钟)
   - Git commit & push
   - 更新 GitHub Releases
   - 同步社交媒体

### 下午（15:00-16:00）

1. **社区互动** (30 分钟)
   - 回复 Issues
   - 审查 PRs
   - 社区答疑

2. **内容优化** (30 分钟)
   - 根据反馈优化内容
   - 修复文档错误
   - 添加新示例

## 🔄 项目协同策略

### 内容互相引流

```
openclaw-recommended-skills (每日更新)
         ↓ 推荐 Skill
openclaw-starter (场景配置)
         ↓ 使用 Skill
openclaw-workflows (工作流)
         ↓ 实际应用
用户增长 ← 社区贡献
```

### README 互相引用

每个项目的 README 都包含：
- 其他两个项目的链接
- 项目矩阵说明
- 协同使用指南

### 交叉推广

- 发布新 Skill → 同时更新 starter 配置
- 发布新工作流 → 同时推荐所需 Skill
- 发布新场景 → 同时推荐相关工作流
