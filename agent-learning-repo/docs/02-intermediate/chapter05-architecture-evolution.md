# Chapter 5: Agent 架构演进

## 5.1 Agent 架构发展历程

### 第一代：简单 LLM 调用

```
┌─────────┐      ┌─────┐      ┌─────────┐
│  输入   │  →   │ LLM │  →   │  输出   │
└─────────┘      └─────┘      └─────────┘
```

**特点：**
- 最简单的架构
- 单次 LLM 调用
- 无状态、无记忆
- 无法使用工具

**适用场景：**
- 简单问答
- 文本生成
- 格式转换

---

### 第二代：增强 LLM（Augmented LLM）

```
┌─────────┐      ┌─────────────────────────┐      ┌─────────┐
│  输入   │  →   │     增强 LLM            │  →   │  输出   │
└─────────┘      │  ┌─────┐ ┌─────┐ ┌────┐ │      └─────────┘
                 │  │检索 │ │工具 │ │记忆│ │
                 │  └─────┘ └─────┘ └────┘ │
                 └─────────────────────────┘
```

**新增能力：**
- **检索（Retrieval）**：访问外部知识库
- **工具（Tools）**：调用外部工具
- **记忆（Memory）**：存储和检索信息

**代表框架：**
- LangChain
- LlamaIndex

---

### 第三代：工作流 Agent（Workflow Agent）

**定义：** 通过预定义的代码路径编排 LLM 和工具的系统。

```
┌─────────────────────────────────────────────────────────┐
│                    工作流 Agent                          │
├─────────────────────────────────────────────────────────┤
│  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐  │
│  │步骤1│ → │步骤2│ → │步骤3│ → │步骤4│ → │步骤5│  │
│  └─────┘    └─────┘    └─────┘    └─────┘    └─────┘  │
│     ↓           ↓           ↓           ↓           ↓   │
│  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐  │
│  │ LLM │    │工具 │    │ LLM │    │ LLM │    │输出 │  │
│  └─────┘    └─────┘    └─────┘    └─────┘    └─────┘  │
└─────────────────────────────────────────────────────────┘
```

**特点：**
- 预定义的执行路径
- 可预测的流程
- 适合结构化任务

**适用场景：**
- 数据处理流水线
- 报告生成
- 客服对话流程

---

### 第四代：自主 Agent（Autonomous Agent）

**定义：** LLM 动态控制自己的执行流程和工具使用的系统。

```
┌─────────────────────────────────────────────────────────┐
│                   自主 Agent                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              ┌─────────────────────┐                    │
│              │      LLM 决策器      │                    │
│              └──────────┬──────────┘                    │
│                         ↓                               │
│    ┌──────────────────────────────────────────────┐    │
│    │              可选行动                         │    │
│    │  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐        │    │
│    │  │工具1│  │工具2│  │工具3│  │思考 │        │    │
│    │  └─────┘  └─────┘  └─────┘  └─────┘        │    │
│    └──────────────────────────────────────────────┘    │
│                         ↓                               │
│              ┌─────────────────────┐                    │
│              │     观察结果         │                    │
│              └──────────┬──────────┘                    │
│                         ↓                               │
│              ┌─────────────────────┐                    │
│              │   继续/终止决策      │                    │
│              └─────────────────────┘                    │
└─────────────────────────────────────────────────────────┘
```

**特点：**
- 动态决策
- 自主规划
- 自我调节
- 适合开放性任务

**代表框架：**
- AutoGPT
- LangGraph
- Microsoft AutoGen

---

## 5.2 架构对比

| 架构类型 | 复杂度 | 灵活性 | 可预测性 | 适用场景 |
|----------|--------|--------|----------|----------|
| 简单 LLM | 低 | 低 | 高 | 简单问答 |
| 增强 LLM | 中 | 中 | 高 | 知识问答 |
| 工作流 Agent | 中 | 中 | 中 | 结构化任务 |
| 自主 Agent | 高 | 高 | 低 | 开放性任务 |

---

## 5.3 架构演进的关键驱动因素

### 1. LLM 能力提升

```
GPT-2 (2019) → GPT-3 (2020) → GPT-4 (2023) → ...
  ↓              ↓              ↓
简单文本      复杂推理      工具使用、规划
```

### 2. 工具生态成熟

```
工具使用能力：
- 代码执行
- API 调用
- 文件操作
- 网络搜索
- 数据库查询
```

### 3. 应用需求升级

```
简单问答 → 复杂任务 → 多步骤工作流 → 自主完成任务
```

## 5.4 现代 Agent 架构示例

### LangChain Agent 架构

```python
from langchain.agents import create_agent

# 创建 Agent
agent = create_agent(
    llm="gpt-4",
    tools=[search_tool, calculator_tool, code_tool],
    memory=ConversationBufferMemory(),
    prompt=AGENT_PROMPT
)

# 执行任务
result = agent.run("帮我分析最近的股票趋势")
```

### Microsoft AutoGen 架构

```python
from autogen import AssistantAgent, UserProxyAgent

# 创建 Agent
assistant = AssistantAgent("assistant", llm_config=llm_config)
user_proxy = UserProxyAgent("user_proxy", code_execution_config=exec_config)

# 多 Agent 协作
user_proxy.initiate_chat(
    assistant,
    message="帮我写一个数据分析脚本"
)
```

## 5.5 架构选择指南

### 何时使用简单 LLM

- ✅ 简单问答
- ✅ 文本生成
- ✅ 格式转换
- ✅ 快速原型

### 何时使用增强 LLM

- ✅ 需要外部知识
- ✅ 需要简单工具
- ✅ 需要短期记忆

### 何时使用工作流 Agent

- ✅ 结构化任务
- ✅ 可预测流程
- ✅ 多步骤处理

### 何时使用自主 Agent

- ✅ 开放性任务
- ✅ 需要动态规划
- ✅ 复杂决策
- ✅ 自主探索

---

## 5.6 小结

Agent 架构经历了四个阶段的演进：

| 阶段 | 架构 | 关键特征 |
|------|------|----------|
| 第一代 | 简单 LLM | 单次调用 |
| 第二代 | 增强 LLM | 工具+检索+记忆 |
| 第三代 | 工作流 Agent | 预定义路径 |
| 第四代 | 自主 Agent | 动态决策 |

选择合适的架构取决于：
- 任务复杂度
- 灵活性需求
- 可预测性要求
