# Chapter 3: 从 Chatbot 到 Agent

## 3.1 Chatbot 的演进历程

### 早期 Chatbot（1960s-1990s）

**ELIZA（1966）**
- MIT 开发
- 基于模式匹配
- 模拟心理治疗师

**特点：**
- 简单的关键词匹配
- 无真正理解能力
- 固定的回复模板

```
用户：我感觉很难过
ELIZA：为什么你觉得很难过？

用户：我不知道
ELIZA：你为什么不知道？
```

---

### 规则驱动 Chatbot（1990s-2010s）

**AIML（Artificial Intelligence Markup Language）**
- 基于 XML 的对话规则
- 人工编写大量规则
- 适用于特定场景

**特点：**
- 需要大量人工规则
- 覆盖面有限
- 维护成本高

```xml
<category>
  <pattern>你好</pattern>
  <template>你好！有什么可以帮你的？</template>
</category>
```

---

### 统计学习 Chatbot（2010s-2020s）

**Seq2Seq 模型**
- 基于 RNN/LSTM
- 端到端学习
- 从对话数据中学习

**特点：**
- 可以生成多样回复
- 但缺乏一致性
- 容易产生"万能回复"

---

### 大语言模型 Chatbot（2020s-至今）

**GPT 系列 / Claude 系列**
- 基于 Transformer
- 海量预训练数据
- 强大的理解和生成能力

**特点：**
- 强大的语言理解
- 灵活的回复生成
- 知识面广
- 但仍有局限性

---

## 3.2 Chatbot 的局限性

### 1. 被动响应模式

```
传统 Chatbot：
用户 → 提问 → Chatbot → 回答 → 结束

问题：
- 只能被动等待用户输入
- 无法主动执行任务
- 无法跨步骤思考
```

### 2. 缺乏工具使用能力

| 能力 | Chatbot | Agent |
|------|---------|-------|
| 搜索信息 | ❌ 无法实时搜索 | ✅ 可调用搜索工具 |
| 执行代码 | ❌ 无法运行代码 | ✅ 可执行代码 |
| 调用 API | ❌ 无法调用外部接口 | ✅ 可调用各种 API |
| 文件操作 | ❌ 无法读写文件 | ✅ 可操作文件系统 |

### 3. 缺乏长期记忆

```
Chatbot 记忆：
- 仅限当前会话
- 会话结束 = 记忆丢失
- 无法学习和积累

Agent 记忆：
- 短期记忆（当前会话）
- 长期记忆（持久化存储）
- 可以从经验中学习
```

### 4. 缺乏规划能力

```
Chatbot：
用户："帮我订机票"
Chatbot："好的，请告诉我具体信息"（只能逐步询问）

Agent：
用户："帮我订机票"
Agent → 分析意图 → 搜索航班 → 比较价格 → 推荐方案 → 自动预订
```

## 3.3 Agent 的进化

### 从 Chatbot 到 Agent 的关键跃迁

```
Chatbot                    Agent
─────────────────────────────────────────────────
被动响应          →        主动执行
单轮对话          →        多轮协作
无工具           →        丰富工具
无记忆           →        长期记忆
无规划           →        复杂规划
```

### 关键技术突破

#### 1. Tool Use（工具使用）

```
Agent 可以：
- 调用搜索引擎
- 执行代码
- 访问数据库
- 调用 API
- 操作文件
```

**代码示例：**
```python
# Agent 工具调用
tools = {
    "search": web_search,
    "calculator": calculate,
    "code_executor": run_code,
    "file_reader": read_file
}

# Agent 可以根据需要选择工具
response = agent.run("搜索最新 AI 新闻并总结")
# → 自动调用 web_search → 总结 → 返回结果
```

#### 2. ReAct 模式（Reasoning + Acting）

```
Thought: 我需要搜索天气信息
Action: web_search("上海明天天气")
Observation: "上海明天阴转小雨，12-18°C"
Thought: 我需要提醒用户带伞
Final Answer: 明天上海有雨，记得带伞！
```

#### 3. Chain-of-Thought（思维链）

```
复杂问题 → 分解步骤 → 逐步推理 → 得出结论

示例：
问题：如果一列火车从北京出发，以 200km/h 的速度行驶...
Step 1: 计算距离
Step 2: 计算时间
Step 3: 考虑中途停靠
Step 4: 得出最终答案
```

#### 4. Planning（规划）

```
目标：帮我准备一个技术分享

Agent 规划：
1. 确定主题和受众
2. 搜集相关资料
3. 制定大纲
4. 制作幻灯片
5. 准备示例代码
6. 演练和优化
```

## 3.4 Chatbot vs Agent 对比

| 维度 | Chatbot | Agent |
|------|---------|-------|
| **主动性** | 被动响应 | 主动执行 |
| **目标导向** | 无明确目标 | 有明确任务目标 |
| **工具使用** | 有限 | 丰富 |
| **记忆** | 会话级 | 长期 |
| **规划** | 无 | 复杂规划 |
| **自主性** | 低 | 高 |
| **适用场景** | 问答、闲聊 | 任务执行、自动化 |

---

## 3.5 何时选择 Chatbot vs Agent

### 选择 Chatbot 的场景

- ✅ 简单问答
- ✅ 闲聊
- ✅ 信息查询
- ✅ 快速回复

### 选择 Agent 的场景

- ✅ 复杂任务执行
- ✅ 多步骤工作流
- ✅ 需要工具使用
- ✅ 需要长期记忆
- ✅ 自动化任务

---

## 3.6 小结

从 Chatbot 到 Agent，是 AI 系统从"对话工具"到"智能助手"的进化：

| 进化阶段 | 关键特征 |
|----------|----------|
| 早期 Chatbot | 模式匹配 |
| 规则 Chatbot | 人工规则 |
| 统计 Chatbot | 数据驱动 |
| LLM Chatbot | 理解+生成 |
| **Agent** | **理解+决策+执行+学习** |
