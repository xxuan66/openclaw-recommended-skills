# 第 3 章 进化什么？(What to Evolve?) - 详细方法分析

## 3.1 模型进化 (Models)

### 3.1.1 核心洞察

> "Models constitute a primary locus of self-evolution, as their parameters can be autonomously rewritten based on the agent's own trajectories, reflections, and interaction outcomes."

**翻译**：模型构成自进化的**主要位点**，因为其参数可以基于智能体自身的轨迹、反思和交互结果自主重写。

**关键能力**：
- 持续适应内部参数
- 扩展功能能力
- 对开发自主通用智能体至关重要

**与静态系统的对比**：

| 方面 | 静态系统 | 自进化模型 |
|------|----------|------------|
| **数据来源** | 依赖人工标注数据集 | 通过交互、自监督数据生成 |
| **训练机制** | 固定训练方案 | 动态学习循环 |
| **效率** | 低（需要人工策划） | 高（自主生成） |
| **适应性** | 有限 | 强 |
| **可扩展性** | 受限 | 高 |

### 3.1.2 模型进化的主要轴线

论文明确了模型展开的三个主要轴线：

#### 轴线 1：从自生成监督学习 (Learning from Self-Generated Supervision)

**目标**：优化模型权重

**代表方法**：

**1. Self-Challenging Agent (SCA)** - Zhou et al. (2025e)

> "One representative approach is the Self-Challenging Agent (SCA), where a language model alternates roles between a challenger generating executable Code-as-Task problems and an executor solving them. The model then fine-tunes its parameters using trajectories from both roles."

**翻译**：一个代表性方法是**自挑战智能体 (SCA)**，其中语言模型在两个角色之间交替：
- **挑战者**：生成可执行的"代码即任务"问题
- **执行者**：解决这些问题

然后模型使用来自两个角色的轨迹微调其参数。

**工作机制**：
```
┌──────────────────────────────────────────┐
│          Self-Challenging Agent          │
├──────────────────────────────────────────┤
│  挑战者角色                               │
│  ↓                                       │
│  生成 Code-as-Task 问题                    │
│  ↓                                       │
│  执行者角色                               │
│  ↓                                       │
│  解决问题                                 │
│  ↓                                       │
│  使用轨迹微调参数                          │
│  ↓                                       │
│  返回挑战者角色 (循环)                     │
└──────────────────────────────────────────┘
```

**优势**：
- 自主生成高质量训练数据
- 覆盖传统基准测试无法覆盖的场景
- 持续自我改进

**2. RAGEN** - Wang et al. (2025p)

**核心技术**：自生成监督 + 环境交互

**进化维度**：
- 策略进化 ✓
- 经验进化 ✓
- 提示进化 ✓
- 单智能体架构优化 ✓

#### 轴线 2：通过与构建或外部环境交互进化 (Evolving Through Interaction)

**目标**：从真实交互中学习

**代表方法**：

**Voyager** - Wang et al. (2023a)

**应用场景**：Minecraft 游戏环境

**进化维度**（全维度进化）：
- 提示进化 ✓
- 记忆进化 ✓
- 工具创建 ✓
- 工具掌握 ✓
- 工具选择 ✓
- 多智能体架构 ✓

**工作机制**：
1. 自主探索游戏环境
2. 遇到新任务/挑战
3. 尝试解决并记录轨迹
4. 从成功/失败中学习
5. 更新技能库和策略

**关键创新**：
- **自主技能获取**：无需人工标注
- **渐进式学习**：从简单到复杂任务
- **技能复用**：已学技能可用于新任务

#### 轴线 3：整合直接重塑未来推理行为的反馈信号 (Integrating Feedback Signals)

**目标**：将反馈直接转化为行为改进

**反馈类型**：
- **执行反馈**：任务成功/失败
- **质量反馈**：输出质量评分
- **对比反馈**：多个输出的偏好排序
- **文本反馈**：自然语言形式的建议

### 3.1.3 模型进化方法总结

| 方法 | 策略进化 | 经验进化 | 核心技术 | 应用场景 |
|------|---------|---------|----------|----------|
| **SCA** | ✓ | ✓ | 自挑战循环 | 代码生成 |
| **RAGEN** | ✓ | ✓ | 自生成监督 + 环境交互 | 通用任务 |
| **Voyager** | - | - | 环境交互 + 技能获取 | Minecraft |

## 3.2 上下文进化 (Context)

### 3.2.1 记忆进化 (Memory Evolution)

**定义**：
> "Context {C_i} – including instructions and long-term memory – which evolves as agents reflect, store, and retrieve experience in ways that shape future decision-making."

**翻译**：上下文 {C_i}——包括指令和长期记忆——随着智能体反思、存储和检索经验而进化，以塑造未来决策。

**记忆进化的核心功能**：
1. **反思 (Reflect)**：从经验中提取洞察
2. **存储 (Store)**：持久化有价值的信息
3. **检索 (Retrieve)**：在需要时获取相关信息

#### 代表方法

**1. Expel** - Zhao et al. (2024a)

**核心技术**：从经验中提取洞察

**进化维度**：
- 经验进化 ✓
- 记忆进化 ✓

**工作机制**：
```
经验收集 → 洞察提取 → 规则形成 → 记忆存储
```

**关键特点**：
- 从具体经验中抽象出通用规则
- 支持跨任务知识迁移
- 渐进式记忆库构建

**2. Agent Workflow Memory** - Wang et al. (2024j)

**核心技术**：工作流级记忆存储

**进化维度**：
- 记忆进化 ✓
- 工具选择 ✓

**创新点**：
- 存储整个工作流而非单个动作
- 支持复杂任务的复用
- 层级化记忆组织

**3. Mem0** - Chhikara et al. (2025)

**核心技术**：个性化记忆管理

**进化维度**：
- 记忆进化 ✓

**应用场景**：
- 个性化助手
- 长期用户交互
- 偏好学习

### 3.2.2 提示优化 (Prompt Optimization)

# 第 4 章 何时进化？(When to Evolve)

## 4.1 测试时内自进化 (Intra-Test-Time Self-Evolution)

### 4.1.1 定义

**测试时内自进化**：在**单个任务执行过程中**进行进化。

**核心特点**：
- **快速、即时**：进化在任务执行期间发生
- **临时、任务特定**：改进通常仅对当前任务有效
- **低延迟要求**：需要在实时约束下完成
- **适用场景**：实时适应、快速调整

### 4.1.2 学习范式

#### In-Context Learning (ICL)

**机制**：通过提示上下文适应

**特点**：
- 无需参数更新
- 利用上下文中的示例
- 即时适应新任务

**优势**：
- 快速部署
- 无训练成本
- 灵活适应

**局限**：
- 上下文长度限制
- 临时性改进
- 依赖示例质量

#### 测试时监督微调 (Test-Time SFT)

**机制**：在测试时快速微调

**特点**：
- 少量梯度更新
- 针对特定任务
- 平衡速度与效果

**应用场景**：
- 高价值任务
- 有即时反馈
- 可接受短暂延迟

#### 测试时强化学习 (Test-Time RL)

**机制**：基于奖励的即时调整

**特点**：
- 从交互中学习
- 试错优化
- 适应动态环境

**挑战**：
- 样本效率
- 探索 - 利用权衡
- 安全性保障

### 4.1.3 测试时内进化方法对比

| 范式 | 速度 | 持久性 | 计算成本 | 适用场景 |
|------|------|--------|----------|----------|
| **ICL** | 快 | 临时 | 低 | 快速适应 |
| **SFT** | 中 | 中期 | 中 | 重要任务 |
| **RL** | 慢 | 中期 | 高 | 动态环境 |

## 4.2 测试时际自进化 (Inter-Test-Time Self-Evolution)

### 4.2.1 定义

**测试时际自进化**：在**任务之间**进行进化，跨任务累积改进。

**核心特点**：
- **渐进、累积**：改进随时间累积
- **持久、跨任务**：效果持续到未来任务
- **可接受批处理**：可在任务间隙进行
- **适用场景**：长期改进、能力构建

### 4.2.2 学习范式

#### In-Context Learning (跨任务)

**机制**：跨任务上下文积累

**特点**：
- 记忆跨任务经验
- 逐步丰富上下文
- 支持知识迁移

**实现方式**：
- 经验库构建
- 案例检索
- 模式提取

#### 监督微调 (跨任务 SFT)

**代表方法**：SCA, RAGEN

**机制**：周期性模型更新

**特点**：
- 批量数据训练
- 持久参数改进
- 支持大规模更新

**工作流程**：
```
任务执行
    ↓
收集轨迹数据
    ↓
积累到阈值
    ↓
批量微调
    ↓
更新模型参数
```

#### 强化学习 (跨任务 RL)

**机制**：长期策略优化

**特点**：
- 跨任务奖励累积
- 策略持续改进
- 支持长期规划

**优势**：
- 全局优化
- 发现长期策略
- 适应复杂环境

### 4.2.3 测试时际进化方法对比

| 范式 | 速度 | 持久性 | 计算成本 | 适用场景 |
|------|------|--------|----------|----------|
| **ICL** | 快 | 中期 | 低 | 知识积累 |
| **SFT** | 中 | 长期 | 中 | 能力提升 |
| **RL** | 慢 | 长期 | 高 | 策略优化 |

## 4.3 时间维度对比总结

| 维度 | 测试时内 (Intra-Test-Time) | 测试时际 (Inter-Test-Time) |
|------|---------------------------|---------------------------|
| **进化速度** | 快速、即时 | 渐进、累积 |
| **持久性** | 临时、任务特定 | 持久、跨任务 |
| **计算成本** | 低延迟要求 | 可接受批处理 |
| **适用场景** | 实时适应 | 长期改进 |
| **典型方法** | ICL、快速 SFT | 批量 SFT、RL |
| **改进类型** | 行为调整 | 参数更新 |
| **知识保留** | 有限 | 充分 |

### 选择指南

**选择测试时内进化当**：
- 需要即时适应
- 任务环境快速变化
- 计算资源受限
- 临时改进足够

**选择测试时际进化当**：
- 追求长期能力提升
- 有充足训练时间
- 需要持久改进
- 跨任务知识迁移重要

# 第 5 章 如何进化？(How to Evolve)

## 5.1 基于奖励的自进化 (Reward-based Self-Evolution)

### 5.1.1 奖励类型详解

#### 文本反馈 (Textual Feedback)

**来源**：人类反馈、LLM 评估

**形式**：
- 自然语言建议
- 错误分析
- 改进指导

**优势**：
- 信息丰富
- 可解释性强
- 支持复杂指导

**挑战**：
- 需要解析
- 可能模糊
- 处理成本高

**代表方法**：
- TextGrad：使用文本梯度优化
- Reflexion：自我反思文本反馈

#### 内部奖励 (Internal Rewards)

**来源**：智能体内部信号

**类型**：
- **自置信度**：模型对输出的自信程度
- **一致性**：多次推理结果的一致性
- **熵**：输出分布的不确定性

**优势**：
- 无需外部信号
- 实时可用
- 低成本

**局限**：
- 可能不准确
- 需要校准
- 易受偏见影响

#### 外部奖励 (External Rewards)

**来源**：外部环境

**类型**：
- **任务成功率**：二元成功/失败信号
- **执行结果**：代码执行结果、API 返回
- **用户反馈**：显式评分、隐式行为

**优势**：
- 客观准确
- 直接相关
- 易于量化

**挑战**：
- 可能稀疏
- 获取成本高
- 延迟反馈

#### 隐式奖励 (Implicit Rewards)

**来源**：从轨迹中推断

**类型**：
- **轨迹质量**：动作序列的流畅性
- **效率指标**：完成时间、步骤数
- **资源使用**：计算资源、API 调用次数

**优势**：
- 无需显式标注
- 自动获取
- 多维度评估

**挑战**：
- 需要设计指标
- 可能冲突
- 权重分配困难

### 5.1.2 奖励设计原则

1. **稠密性**：提供频繁的反馈信号
2. **一致性**：与最终目标对齐
3. **可计算性**：高效计算
4. **鲁棒性**：抵抗噪声和异常值
5. **可解释性**：理解奖励来源

---

## 5.2 模仿与示范学习 (Imitation and Demonstration Learning)

### 5.2.1 自生成示范学习 (Self-Generated Demonstration Learning)

**定义**：智能体生成自己的训练数据并从中学习。

**工作机制**：
```
自主探索
    ↓
生成轨迹
    ↓
自我评估
    ↓
筛选优质示范
    ↓
加入训练集
    ↓
微调模型
```

**优势**：
- 无需人工标注
- 覆盖长尾场景
- 持续扩展数据

**代表方法**：
- SCA：自挑战生成问题 - 解决对
- Voyager：自主探索生成技能

### 5.2.2 跨智能体示范学习 (Cross-Agent Demonstration Learning)

**定义**：从其他智能体学习。

**学习方式**：
1. **观察学习**：观察其他智能体的行为
2. **知识蒸馏**：从更强智能体学习
3. **协作学习**：多智能体互相学习

**优势**：
- 加速学习
- 知识迁移
- 避免重复探索

**挑战**：
- 智能体对齐
- 知识兼容性
- 信用分配

### 5.2.3 混合示范学习 (Hybrid Demonstration Learning)

**定义**：结合自生成和外部示范。

**组合策略**：
- **预训练 + 自改进**：先用外部数据预训练，再自生成改进
- **并行学习**：同时从自生成和外部数据学习
- **课程学习**：从外部示范开始，逐渐转向自生成

**优势**：
- 结合两者优点
- 平衡质量和多样性
- 灵活适应

---

## 5.3 基于群体和进化方法 (Population-based and Evolutionary Methods)

### 5.3.1 单智能体进化 (Single Agent Evolution)

**机制**：变异和选择

**进化算法框架**：
```
初始化种群
    ↓
评估适应度
    ↓
选择
    ↓
变异/交叉
    ↓
新一代种群
    ↓
(重复直到收敛)
```

**应用对象**：
- 提示模板
- 工作流结构
- 超参数配置

**代表方法**：
- Promptbreeder：提示进化
- AlphaEvolve：架构进化

### 5.3.2 多智能体进化 (Multi-Agent Evolution)

**机制**：群体智能 + 协同进化

**进化模式**：
1. **竞争进化**：智能体间竞争，优胜劣汰
2. **协作进化**：智能体协作，共同改进
3. **生态进化**：模拟生态系统，多样性保持

**优势**：
- 探索更广阔空间
- 发现协同模式
- 鲁棒性强

**挑战**：
- 计算成本高
- 协调复杂
- 收敛慢

---

## 5.4 交叉进化维度 (Cross-cutting Evolutionary Dimensions)

### 5.4.1 在线与离线学习 (Online and Offline Learning)

| 维度 | 在线学习 | 离线学习 |
|------|---------|---------|
| **数据来源** | 实时交互 | 预收集数据 |
| **更新频率** | 持续 | 批量 |
| **适应性** | 强 | 弱 |
| **稳定性** | 弱 | 强 |
| **适用场景** | 动态环境 | 静态环境 |

### 5.4.2 同策略与异策略学习 (On-policy and Off-policy Learning)

| 维度 | 同策略 (On-policy) | 异策略 (Off-policy) |
|------|-------------------|-------------------|
| **数据来源** | 当前策略 | 历史策略 |
| **样本效率** | 低 | 高 |
| **稳定性** | 高 | 低 |
| **探索能力** | 有限 | 强 |
| **代表算法** | PPO, TRPO | DQN, SAC |

### 5.4.3 奖励粒度 (Reward Granularity)

| 粒度 | 描述 | 优势 | 挑战 |
|------|------|------|------|
| **稀疏奖励** | 任务级信号 | 简单、清晰 | 信用分配难 |
| **稠密奖励** | 步骤级信号 | 指导性强 | 设计复杂 |
| **分层奖励** | 多粒度组合 | 平衡 | 权重调优 |

## 5.5 进化方法选择指南

### 根据任务特性选择

| 任务特性 | 推荐方法 | 理由 |
|----------|---------|------|
| **有明确奖励** | 奖励基 | 直接优化 |
| **示范可用** | 模仿学习 | 利用先验知识 |
| **探索空间大** | 群体进化 | 广泛搜索 |
| **实时要求高** | 在线学习 | 即时适应 |
| **数据稀缺** | 自生成 + 离线 | 数据增强 |

### 根据资源约束选择

| 资源约束 | 推荐方法 | 理由 |
|----------|---------|------|
| **计算充足** | 群体进化 + 在线 | 充分探索 |
| **计算受限** | 奖励基 + 离线 | 高效利用 |
| **时间充裕** | 异策略 + 稠密奖励 | 精细优化 |
| **时间紧迫** | 同策略 + 稀疏奖励 | 快速收敛 |
